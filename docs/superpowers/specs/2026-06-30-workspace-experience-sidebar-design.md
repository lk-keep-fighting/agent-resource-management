# 工作空间右侧：工作经验侧栏（浏览 / 搜索 / 引用）

- 日期：2026-06-30
- 作者：刘锟 + Claude
- 状态：设计定稿，待评审
- 关联：`docs/superpowers/specs/2026-06-30-knowledge-binding-dimension-design.md`（知识飞轮 v1：essential/experience 维度）、`AGENTS.md`

## 1. 背景与动机

知识飞轮 v1 已给 `AgentKnowledgeBinding` 增加 `kind` 维度：

- `essential`（必备业务知识）：运行时**下载到 Agent 工作区文件**，始终可读。
- `experience`（工作经验）：**不下载**，Agent 排障时按需检索（`knowledge_search` 工具）；规模可无限增长。

当前问题：`experience` 只在 Agent 主动检索时才被用到，**人类用户在对话过程中看不到、也无法主动引用**。用户希望在工作空间右侧随时看到当前 Agent（"员工"）关联的最新工作经验，便于：

1. **获取**——及时翻阅相关踩坑经验，心里有数；
2. **引用**——把某条经验直接带进当前对话，让 AI 在本轮就能感知到，而不是干等 Agent 自己去检索。

本设计即在工作空间右侧侧栏增加一个"工作经验"面板，支撑浏览、搜索、引用三个动作，闭合飞轮的"使用"环节。

## 2. 目标与非目标

### 目标
1. 在工作空间右侧 `.chat-side` 增加工作经验区：默认展示最新经验，支持搜索、阅读全文。
2. 提供"引用"动作：把选中经验的全文作为**本轮附加上下文**注入对话，AI 立即可见。
3. 顺带支持查看本 Agent 已加载的必备知识（折叠区），知情即可。
4. 复用现有数据通路与 UI 构件，改动最小化。

### 非目标（明确不做）
- 改变 `experience` 的运行时下载策略（仍不下载到工作区）。
- 经验的编辑 / 上传 / 绑定 UI（仍在控制台 `agents/page.tsx`；工作空间侧只读）。
- RAG 语义检索（搜索 v1 用关键字过滤；语义检索留飞轮后续阶段）。
- 经验的评分 / 反馈 UI（复用现有反馈机制，不在本面板内做）。
- 跨 Agent 的经验聚合（仅展示当前工作空间所绑定 Agent 的经验）。

## 3. 核心交互设计

前端落点：`workstation/public/main.js` 的 `renderWorkspaceChat(workspaceId)`，侧栏元素 `side`（`.chat-side`，当前约 1457–1471 行）。

### 3.1 放置与面板重构

当前 `.chat-side`（`styles.css` 约 269 行，`height: fit-content`）：Agent 头像 → `kv`（Agent/Version/WS）→ 分隔线 → 📌 上下文。

改造后结构（在「📌 上下文」之下追加）：

```
.chat-side (position: sticky，跟随对话滚动)
├─ Agent 头像 + 名称
├─ kv (Agent / Version / WS)
├─ divider
├─ 📌 上下文 (可折叠，保留)
├─ divider
└─ 📚 工作经验 (新增，主体)
   ├─ 已加载必备 N 篇 (折叠区，默认收起)
   ├─ 标题行：📚 工作经验  + 数量徽标 + 🔍
   ├─ 搜索框 (点 🔍 展开)
   └─ 经验列表 (overflow-y:auto，内部滚动)
```

要点：
- `.chat-side` 改 `position: sticky`，Agent 信息固定顶部；
- 经验列表区独立 `overflow-y: auto; max-height: calc(...)`，**列表内部滚动**，不撑爆页面；
- 复用 `el()` / `.divider` / `.muted` / `.kv`；新增 class 见 §3.7。

### 3.2 经验列表（默认态）

- **"最新"定义**：按知识 `updatedAt` 倒序（内容最近更新/创建的在前）。
- **默认展示**：最新 **8 条**；超出时底部提示「共 N 条 · 🔍 搜索全部」。
- **每条卡片**（紧凑行）：
  - 标题（知识 `name`）
  - 一行描述（`description` 截断，`.muted`）
  - 元信息：相对时间（「2 天前更新」，基于 `updatedAt`）+ 作者
  - 绿色左条标示"经验"类型
  - 点击卡片 → 打开阅读抽屉（§3.4）
  - 右侧 `📌` 小图标 → 一键引用（不开抽屉的快捷路径，§3.5）
- 空闲态 / 加载态 / 错误态见 §4。

### 3.3 已加载必备知识（折叠区）

- 经验区顶部一行可折叠：`📦 已加载必备 N 篇`（默认收起）。
- 展开后列出 `essential` 绑定的知识名（只读，标注"已下载到工作区"）。
- 点击某条同样打开阅读抽屉查看全文；必备条目用**琥珀色**徽标与经验（绿色）区分。
- 目的：让用户知情"这位员工上岗必修了哪些"，零额外操作成本。

### 3.4 阅读抽屉

- 从右侧滑出，**覆盖对话区**（侧栏保留可见），宽 `min(640px, 60vw)`，带半透明遮罩。
- 内容：
  - 头部：标题 + 类型徽标（经验=绿 / 必备=琥珀）+ ✕
  - 元信息：更新时间 / 作者 / version
  - 正文：`renderMarkdown()` 渲染全文（独立滚动）
  - 底部：`[复制全文]`（次）；主按钮按类型区分——
    - **经验**：`[📌 引用进对话]`（§3.5）
    - **必备**：显示 `✓ 已在工作区`（内容已下载到 Agent 环境，无需重复引用；必备条目为只读查看）
- 交互：`Esc` / 点遮罩关闭；关闭后焦点回到列表。
- 数据：打开时按需 `GET /knowledges/:id` 拉全文（列表只持元信息，懒加载）。

### 3.5 引用流程（核心交互：标签附加，按消息清空）

- 点「📌 引用」（抽屉底部按钮，或列表卡片右侧 `📌` 快捷图标）：
  1. 关闭抽屉（若开着）；
  2. 在**输入框上方**出现一个可移除的绿色标签：`📌 经验名 ✕`；
  3. 允许**多条叠加**；点 `✕` 移除单条。
- **发送时**：
  - 本轮 run 携带已引用经验的 id（`pinnedExperienceIds[]`），由 workstation 解析全文、作为**本轮附加上下文**注入；
  - 标签**按消息清空**——引用一次即已进入该轮上下文，无需持续引用造成上下文爆炸。
- **可见性反馈**：有标签时，输入区下方轻提示「本次对话将附带 N 条经验」，让用户明确知道上下文已挂上。
- **软上限**：引用条数上限 5，超出显示 `+N` 并轻提示，防止上下文爆炸。

### 3.6 搜索

- 点标题行 `🔍` → 下方出现搜索框。
- **范围**：**仅当前 Agent 的经验池**（贴合"选中的员工"）。
- **机制（v1）**：对已拉取的经验元信息做**客户端过滤**（`name` + `description` 命中），即时、零后端改动；未来池子极大时再退回后端 `searchKnowledges({keyword})`。
- **无结果**：「没找到匹配的经验，换个词？」+ 清除按钮；`Esc` / 清除 → 回到最新列表。

### 3.7 视觉语言

- 经验统一**绿色**（对齐控制台 `bg-green-50 / border-green-200`）；必备用**琥珀色**。
- 复用 `.tag / .kv / .divider / .muted`、`renderMarkdown()`、`el()`。
- 新增 CSS class（加在 `styles.css` 的 `.chat-*` 区附近）：`.exp-section / .exp-item / .exp-item:hover / .essential-collapse / .exp-drawer / .exp-drawer-backdrop / .cite-chip / .cite-bar`。

## 4. 边界状态

| 场景 | 处理 |
|---|---|
| 无经验绑定 | 空态「这位员工暂无工作经验沉淀」+ CTA「把这次对话总结成经验」→ 复用现有 `handleSummarize`（接上飞轮的"上传"环节） |
| 加载中 | 骨架行 / spinner |
| 拉取失败 | 「经验加载失败 [重试]」 |
| Agent 未启用工具（`enableTools=false`） | 不影响——经验仍可读可引用；人工引用把内容直接注入上下文，绕过 Agent 的 `knowledge_search` 工具 |
| 引用超过 5 条 | 软上限，显示 `+N`，轻提示 |
| 经验全文为空 | 抽屉正文显示「（无内容）」，仍可引用（注入空内容无害） |
| `GET /knowledges/:id` 失败 | 抽屉显示「内容加载失败 [重试]」；引用按钮在该条禁用 |

## 5. 数据与后端衔接

### 5.1 复用（无改动）
- `GET /agents/:id`（workstation 代理 ARM）→ `knowledgeBindings[]`，每条含 `{knowledgeId, knowledgeName, version, kind}`。
- `GET /knowledges/:id` → 全文 `{id, name, description, content, updatedAt, createdBy}`。
- `searchKnowledges({keyword})`（v1 搜索的退路）。

### 5.2 前端（`workstation/public/main.js`）
- `renderWorkspaceChat` 内：渲染工作经验区、必备折叠区、阅读抽屉、引用标签栏 + 状态（`citedExperienceIds[]`）。
- 列表元信息从 `GET /agents/:id` 的绑定取；全文懒加载。
- 搜索：客户端过滤已持元信息。

### 5.3 核心实质改动：引用上下文注入
当前 send → run 链路不携带"用户本轮指定上下文"。需让：

1. **send / 创建 run 的入口**接受 `pinnedExperienceIds[]`（仅当用户引用时携带）。
2. **`executeRun` / `context-builder`** 接收该数组，按 id 解析全文（复用 essential 的进程内缓存模式），注入一个**本轮专用**的分区：

   ```
   ## 用户本次引用的工作经验
   ### <name1>
   <content1>
   ### <name2>
   <content2>
   ```

   该分区**仅对携带 `pinnedExperienceIds` 的 run 生效**，不改变既有「必备业务知识 / 工作经验（按需检索）」分区（见飞轮 v1 §5.4）。

3. 解析失败（网络/权限）：该条降级为"只列名 + 加载失败提示"，不阻塞 run。

### 5.4 小数据需求
- 列表"最新"排序与相对时间需要 `updatedAt`。若 `GET /agents/:id` 的绑定项当前不回传 `updatedAt` / `createdBy`，需在该接口顺带回传（或在列表渲染时按需 `GET /knowledges/:id` 补齐——但为排序需先拿到，优先让接口回传）。

### 5.5 无 schema 变更
`kind` 字段已存在（飞轮 v1 已落地）。

## 6. 测试策略

- **前端（人工 / 冒烟）**：
  - 有 / 无经验绑定时面板渲染正确；空态 CTA 触发 `handleSummarize`。
  - 列表按 `updatedAt` 倒序、默认 8 条、"共 N 条"提示。
  - 阅读抽屉开/关、`Esc` / 遮罩关闭、全文 markdown 渲染。
  - 引用：加标签、多标签、`✕` 移除、发送后清空、软上限 5。
  - 搜索：客户端过滤命中、无结果态、清除回列表。
  - 必备折叠区展开/收起、琥珀徽标区分。
- **workstation 集成**：构造带 `experience` + `essential` 绑定的 mock agent → 发送一条携带 `pinnedExperienceIds` 的消息 → 断言该轮 system prompt 含「用户本次引用的工作经验」分区且内容正确；不携带时不含该分区。复用 `mock-arm` + e2e 脚本模式。
- **错误路径**：`getKnowledgeById` 失败 → 引用降级为只列名，run 不中断。
- **验证顺序**（遵循 `AGENTS.md`）：workstation 改动 `bun run dev` + 必要时 `mock-arm` + e2e；前端冒烟。

## 7. 遵守的约定
- 引用上下文是**派生数据**（按 id 现取现注入），不落库、不破坏 append-only 版本化绑定。
- API 响应沿用 `{ok, data, msg}`；认证沿用 API key / SSO。
- 工作空间侧只读引用，不产生新的绑定 / 版本。
- 不 update / 硬删旧行（本设计无写入操作）。

## 8. 已定项（评审结论）
- [已定] 核心交互：浏览 + 引用 + 搜索（三者皆要）。
- [已定] 引用方式：标签附加（输入框上方可移除标签，发送时把全文作为本轮附加上下文）。
- [已定] 阅读区：面板清单 + 右侧滑出阅读抽屉（覆盖对话区）。
- [已定] 引用标签**按消息清空**（引用一次已入上下文，避免持续引用造成上下文爆炸）。
- [已定] 默认展示最新 **8 条**，其余靠搜索。
- [已定] 侧栏顺带展示**必备知识折叠区**（默认收起），便于用户随时查看。

## 9. 后续阶段（不在本 spec 范围）
- 搜索退回后端 / 语义检索（与飞轮 v1 后续 RAG 阶段合并）。
- 引用经验的反馈回流到飞轮（被引用次数 → 经验质量信号）。
- 多 Agent 经验聚合 / 跨工作空间经验复用。
