# Agent员工管理 - 完整UX设计方案

## 一、设计理念

### 1.1 核心目标
让办公小白用户在**3分钟内**创建一名Agent员工，并为其配置好能力和知识。

### 1.2 设计原则
- **接地气**：用"员工"代替"Agent"，"能力"代替"Skill"，"知识"代替"Knowledge"
- **简洁化**：每一步只做一件事，避免信息过载
- **可发现**：用户能轻松找到"新建"入口，且知道如何创建不存在的资源

## 八、交互细节

### 8.1 必填校验
- 基本信息步骤："员工姓名"为必填
- 能力配置步骤：至少选择一个能力（可选，但建议提示）
- 知识配置步骤：至少选择一份知识（可选，但建议提示）

### 8.2 导航规则
- 新建流程中点击"取消"：弹出确认框"确定要放弃创建吗？"
- 上一步/下一步：根据当前步骤状态按钮
- 最后一步"完成"变为"保存"，成功后显示成功提示

### 8.3 快捷操作
- 选择能力/知识时，点击即选中/取消，无需确认
- 编辑状态下直接修改，无需进入步骤流程
- 删除操作需要二次确认

---

## 九、响应式策略

- **桌面（≥1024px）**：左右分栏布局
- **平板（768-1023px）**：左右分栏，右侧可全屏
- **手机（<768px）**：上下布局，列表和详情切换显示

---

## 十、Accessibility

- 所有输入框有对应 label
- 模态框有 focus trap
- 进度指示器有 aria-current
- 支持键盘导航（Tab, Enter, Escape）
